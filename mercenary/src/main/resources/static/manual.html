2.5 Audit Logging

All security-relevant events are persisted to the audit_log collection:

  • AUTH_FAILURE: Invalid credentials or missing certificates
  • ACCESS_DENIED: Valid user attempting to access restricted data
  • PROMPT_INJECTION: Detected manipulation attempt (e.g., "Ignore previous instructions")
  • DATA_INGESTION: Upload of new intelligence dossiers
  • QUERY_EXECUTED: Every query with sector, user, and response summary

3.1 Ingestion Workflow

  1. Open Control Panel: Click the hamburger menu (‚ò∞) or press Ctrl + U.
  2. Select Target Sector: Choose from the dropdown (e.g., Defense / Military). This determines the clearance required to access this document.
  3. Upload Document: Click the upload zone or drag-and-drop your file. Supported formats: PDF, TXT, MD (max 50MB).
  4. Wait for Processing: The system will parse, chunk, vectorize, and index the document. The telemetry bar will update to show the new document count.
Automatic PII Redaction

Why Glass Box Matters

Most enterprise AI systems operate as "black boxes" ‚Äî you ask a question, you get an answer, but you have no visibility into how that answer was generated. This creates three critical problems:

Problem Black Box Risk Sentinel's Glass Box Solution
Compliance Audits Cannot prove how answers were derived Full chain logged and displayable per response
Hallucination Detection No way to verify if AI fabricated information See exactly which documents were retrieved and cited
Trust & Adoption Users distrust opaque systems Operators can verify every step of the reasoning

The ANALYZE ‚Üí VERIFY ‚Üí CITE Protocol

Sentinel's response synthesis follows a strict three-phase protocol:

  1. ANALYZE: Parse the retrieved document chunks and identify relevant facts.
  2. VERIFY: Cross-check facts against source metadata. Reject unsupported claims.
  3. CITE: Attach source attribution [filename.ext] to every fact in the response.
Compliance Note

For regulated industries (DoD, Finance, Healthcare), Glass Box reasoning provides the audit trail required by frameworks like NIST AI RMF, DoD AI Ethics Principles, and FDA guidance on AI/ML-based software. Every query's reasoning chain can be exported for compliance review.

4.3 Security Enforcement

Every query passes through validation:

  1. Authentication: User must be logged in (or have valid CAC certificate).
  2. Permission Check: User role must include QUERY permission.
  3. Clearance Check: User clearance must meet or exceed sector requirement.
  4. Prompt Injection Detection: Queries containing attack patterns are blocked.

5.1 Deep Storage Recovery

Notice the banner in the screenshot: STATUS: RECONSTRUCTED FROM VECTOR STORE. This indicates the original file was not in cache, so Sentinel reconstructed the content from the vector database.

Recovery Limitations

Deep Storage Recovery retrieves the text chunks stored during ingestion, not the original file. Recovered content may differ from the source in formatting, structure, and completeness. Documents recovered this way are marked with an (ARCHIVE) banner.

7. Advanced RAG Features

Sentinel implements three cutting-edge RAG techniques from recent research to improve retrieval accuracy, security, and reasoning capabilities.

7.1 HiFi-RAG (High-Fidelity Retrieval)

Based on arXiv:2512.22442v1, HiFi-RAG implements iterative two-pass retrieval with cross-encoder reranking:

üîÑ Two-Pass Retrieval

Initial broad retrieval followed by focused refinement based on identified gaps.

‚öñÔ∏è Cross-Encoder Reranking

LLM-as-judge semantic scoring via Ollama for precise relevance ranking.

üîç Gap Detection

Identifies missing concepts and retrieves additional documents to fill knowledge gaps.

‚úàÔ∏è Air-Gap Safe

Falls back to keyword-based scoring when LLM is unavailable.

7.2 RAGPart (Corpus Poisoning Defense)

Based on arXiv:2512.24268v1, RAGPart defends against corpus poisoning attacks:

Feature Description
Document Partitioning SHA-256 based deterministic assignment to multiple partitions
Consistency Scoring Detects anomalous documents via cross-partition agreement analysis
Suspicion Flagging Documents with high variance scores are flagged for security review
Combinatorial Retrieval Queries multiple partition combinations for robustness
Corpus Poisoning Protection

RAGPart automatically detects when an adversary has injected malicious documents into the corpus. Documents that appear inconsistent across partition combinations are flagged with a suspicion score, alerting operators to potential security threats.

7.3 HGMem (Hypergraph Memory)

Based on arXiv:2512.23959v2, HGMem enables multi-hop reasoning across documents:

Component Function
Entity Extraction Pattern-based NER identifies PERSON, ORG, LOCATION, DATE, TECHNICAL entities
Hypergraph Storage MongoDB-backed node and hyperedge collections for relationship storage
Multi-hop Traversal BFS graph exploration discovers relationships across multiple documents
Hybrid Query Combines vector similarity with graph context for comprehensive retrieval

Example use case: "What is the relationship between Project X and Contractor Y?" ‚Äî HGMem can traverse the hypergraph to find connections across multiple documents even if they don't directly mention each other.

8. Troubleshooting

8.1 Connection Issues

Symptom Likely Cause Resolution
SYSTEM OFFLINE (red badge) Ollama not running Start Ollama: ollama serve
Vector DB: Offline MongoDB not running Start MongoDB: mongod
High AVG LATENCY (>5000ms) Insufficient memory Close other apps; ensure 16GB+ RAM
ACCESS DENIED on queries User clearance too low Contact administrator to verify clearance
Upload fails File >50MB or password-protected Split file or remove PDF protection

8.2 Document Issues

  • Password-protected PDFs: Not supported. Remove protection before ingestion.
  • Scanned PDFs (image-only): Limited support. OCR quality varies.
  • Large files (>50MB): Will be rejected. Split into smaller documents.
Critical Error Support

If issues persist after troubleshooting, generate a system log and contact your Deployment Specialist. Include: error messages, timestamp, user ID, and steps to reproduce.

9. Air-Gap Deployment

For disconnected or SCIF environments, Sentinel operates with zero external network dependencies:

  • Local Inference: All reasoning is performed by Ollama on localhost:11434. No data leaves the perimeter.
  • Local Vector Store: All document embeddings stored in MongoDB on localhost:27017.
  • Local Embeddings: Embedding model (nomic-embed-text) runs via Ollama.

9.1 Required Environment Variables

Variable Value Purpose
APP_PROFILE govcloud Activates CAC/PIV authentication
OLLAMA_URL http://localhost:11434 Local LLM inference endpoint
MONGODB_URI mongodb://localhost:27017/mercenary Local vector store

9.2 System Requirements

  • Minimum RAM: 16 GB (32 GB recommended for large models)
  • Storage: 20 GB for application + model storage
  • Network: None required for air-gap deployment
  • Ports: 8080 (application), 11434 (Ollama), 27017 (MongoDB)
