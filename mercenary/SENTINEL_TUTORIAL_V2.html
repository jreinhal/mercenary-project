1. System Overview

Sentinel is an air-gapped Intelligence Platform designed to synthesize answers from secure document repositories without data leakage.

SECURITY PROTOCOL

Air-Gapped Foundation

Sentinel runs entirely on local infrastructure (Llama3 + MongoDB). No data ever touches the cloud. Authentication is enforced via CAC/PIV certificates or OIDC.

INTELLIGENCE

Glass Box Reasoning

Unlike "black box" AI, Sentinel cites its sources. Every fact in a response is anchored to a specific file [mission_report.pdf] for immediate verification.

2. Search Capabilities

FEATURE

Hybrid Search

Sentinel combines two search technologies to maximize recall:

  • Semantic Search: Understands meaning (e.g., "Vehicle" matches "Truck").
  • Keyword Fallback: Scans for exact terms (e.g., "Project Obsidian").

Result: You never miss a document because of slightly different phrasing.

FEATURE

Query Decomposition

The system automatically detects complex questions:

"Who is Captain Vasquez and what is the budget for Project Nexus?"

Action:

  1. Splits into: "Who is Captain Vasquez"
  2. Splits into: "What is budget for Project Nexus"
  3. Retrieves docs for BOTH.
  4. Synthesizes unified answer.

3. Security & Compliance

The 4-Layer Defense

  1. Authentication: Verifies identity (CAC/PIV).
  2. RBAC: Checks permissions (e.g., QUERY vs INGEST).
  3. Clearance: Checks Security Level (SECRET vs UNCLASSIFIED).
  4. Audit: Logs every action to immutable storage.

Prompt Injection Defense

The system actively scans inputs for adversarial attacks (e.g., "Ignore previous instructions"). Malicious queries are blocked and logged as security incidents.

PII Redaction

During ingestion, patterns matching SSNs and Emails are automatically redacted [REDACTED-SSN] to prevent accidental leakage in retrieval.

4. Operator Guide

How to Ingest Data

  1. Navigate to the **Control Panel**.
  2. Select the target **Sector** (e.g., DEFENSE, MEDICAL).
  3. Drag & Drop files or select using the file picker.
  4. Watch the **Telemetry** bar to confirm index count increase.

Best Practices for Querying

  • Be Specific: "Report on Site Echo status" is better than "Report".
  • Use Compound Questions: Feel free to ask multi-part questions connected by "and".
  • Verify Citations: Always click the citations [file.txt] to verify the source context in the Evidence Viewer.
