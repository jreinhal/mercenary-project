Security Architecture

SENTINEL enforces a Zero Trust security model with defense-in-depth controls. Access is validated through multiple phases before any data is returned.

Security Phases

PhaseComponentAction
Phase 1 Transport Security TLS termination, client certificate validation
Phase 2 Identity Verification X.509 DN parsing, user lookup/provisioning
Phase 3 Authorization Role and clearance checks, audit logging
Auto-Provisioning

New CAC users are auto-provisioned with VIEWER role and UNCLASSIFIED clearance. Administrators must manually upgrade clearance after verification.

Clearance Hierarchy

Beyond standard RBAC, SENTINEL enforces Clearance-Based Access Control (CBAC). Even an admin cannot access higher-clearance data without proper authorization.

Audit Logging

All security-relevant events are persisted to the audit log:

  • AUTH_FAILURE: Invalid credentials or certificates
  • ACCESS_DENIED: Insufficient clearance for requested data
  • PROMPT_INJECTION: Detected manipulation attempts
  • DATA_INGESTION: Document uploads
  • QUERY_EXECUTED: Every query with full context

Document Ingestion

Documents must be ingested into the vector store before they can be queried. Use the Files button in the menu bar or press Ctrl + U.

Ingestion Workflow

  1. Select Files: Click Files or drag-and-drop into the upload zone
  2. Choose Sector: Documents inherit the currently active sector
  3. Processing: System parses, chunks, vectorizes, and indexes content
  4. Confirmation: Document appears in Sources panel when ready

Connector Sync (Admin)

Enterprise connector ingestion (S3, SharePoint, Confluence) supports incremental sync behavior. For recurring sync jobs, unchanged sources are skipped, changed sources are re-ingested, and removed sources are pruned from the vector store.

  • Manual Admin Sync: POST /api/admin/connectors/sync
  • Scheduled Sync: enable SENTINEL_CONNECTORS_SYNC_ENABLED=true
  • Incremental Mode: SENTINEL_CONNECTORS_INCREMENTAL_SYNC_ENABLED=true
  • Legacy Backfill: one-time migration via SENTINEL_CONNECTORS_LEGACY_MIGRATION_ENABLED=true (run one connector sync first, then run dry-run)
Automatic PII Redaction

SENTINEL automatically detects and redacts SSNs and email addresses during ingestion. Operators should still pre-screen for credentials and classified markings.

Intelligence Queries

Once documents are ingested, use the chat interface to query the system. SENTINEL retrieves relevant document chunks and synthesizes responses with mandatory citations.

Query Behavior

  • Context Aware: Answers are based strictly on ingested documents
  • Citation Required: Every fact includes a clickable source reference
  • Confidence Score: Shows retrieval quality (higher = more relevant sources)
  • Reasoning Chain: Expandable view of the full retrieval process

The ANALYZE ‚Üí VERIFY ‚Üí CITE Protocol

SENTINEL's response synthesis follows a strict three-phase protocol:

  1. ANALYZE: Parse retrieved chunks and identify relevant facts
  2. VERIFY: Cross-check facts against source metadata
  3. CITE: Attach source attribution to every fact in the response
Compliance Note

Glass Box reasoning provides the audit trail required by NIST AI RMF, DoD AI Ethics Principles, and FDA guidance on AI/ML-based software. Every query's reasoning chain can be exported.

Evidence Verification

SENTINEL enforces strict auditability. Every fact is anchored to a specific source document. Click any citation badge to view the source in the Information Panel.

Information Panel

The right panel displays two tabs:

  • Plot: Interactive knowledge graph showing document relationships
  • Sources: List of retrieved documents with similarity scores

Page and Region Verification

For retained source PDFs, operators can verify evidence at page or region granularity. This supports exact source checks for dense tables, diagrams, and quoted text.

  • Source Page: Render full source page for a cited document
  • Source Region: Render a bounded box with configurable context expansion
  • Policy-Aware Access: Rendering is blocked when retention policy forbids source bytes
Recovery Limitations

Deep Storage Recovery retrieves text chunks from the vector database, not original files. Recovered content may differ in formatting. Documents are marked with (ARCHIVE) status.

Advanced RAG Features

SENTINEL implements cutting-edge RAG techniques from recent research to improve retrieval accuracy, security, and reasoning capabilities.

HiFi-RAG (High-Fidelity Retrieval)

Based on arXiv:2512.22442v1, implements iterative two-pass retrieval:

Two-Pass Retrieval

Cross-Encoder Reranking

Dedicated reranking with automatic fallback modes for stable relevance scoring.

RAGPart (Corpus Poisoning Defense)

Based on arXiv:2512.24268v1, defends against adversarial attacks:

  • Document Partitioning: SHA-256 based deterministic assignment
  • Consistency Scoring: Cross-partition agreement analysis
  • Suspicion Flagging: High-variance documents flagged for review

HGMem (Hypergraph Memory)

Based on arXiv:2512.23959v2, enables multi-hop reasoning:

  • Entity Extraction: Pattern-based NER for key entities
  • Hypergraph Storage: MongoDB-backed relationship storage
  • Multi-hop Traversal: BFS exploration across documents

Domain Thesaurus and Temporal Filtering

Retrieval can expand specialized terms and apply date-aware prefilters when queries include time bounds.

  • Domain Expansion: Acronym/synonym expansion per sector and workspace
  • Unit Normalization: Optional conversion-aware matching (for validated deployments)
  • Temporal Prefilter: Year/range constraints narrow retrieval scope before synthesis

Multimodal and Visual Evidence Support

  • Visual Context: Image and chart metadata can participate in retrieval context
  • Source Rendering: Page/region rendering endpoints support direct evidence validation
  • Regulated Controls: HIPAA-regulated deployments can disable persistent visual retention

Ingestion Resilience and Safety

  • Checkpoint/Resume: Long-running ingestion can recover from interruptions
  • Bounded Retries: Retry logic and fallback modes reduce batch failure rates
  • Failure Thresholds: Safety limits stop ingestion when failure rates exceed configured bounds

Hallucination Risk Controls

  • Uncertainty Analysis: Quantified-Confidence RAG (QuCo-RAG) flags unsupported or low-confidence generations but does not guarantee factual correctness
  • Abstention Path: High-risk responses are suppressed in favor of evidence-first fallback
  • Citation Enforcement: Regulated sectors require strict source-backed responses

Air-Gap Deployment

For disconnected or SCIF environments, SENTINEL operates with zero external network dependencies:

  • Local Inference: All reasoning via Ollama on localhost:11434
  • Local Vector Store: MongoDB on localhost:27017
  • Local Embeddings: nomic-embed-text model via Ollama

Environment Variables

VariableValuePurpose
APP_PROFILE govcloud CAC/PIV authentication
OLLAMA_URL http://localhost:11434 Local LLM endpoint
LLM_MODEL llama3.1:8b Chat model
EMBEDDING_MODEL nomic-embed-text Embedding model
MONGODB_URI mongodb://localhost:27017/mercenary Local vector store

System Requirements

  • RAM: 16 GB minimum (32 GB recommended)
  • Storage: 20 GB for application + models
  • Network: None required for air-gap
  • Ports: 8080, 11434, 27017

Troubleshooting

Connection Issues

SymptomCauseResolution
SYSTEM OFFLINE Ollama not running Run ollama serve
Vector DB: Offline MongoDB not running Run mongod
High Latency (>5s) Insufficient memory Ensure 16GB+ RAM available
ACCESS DENIED Clearance too low Contact administrator
Upload fails File >50MB or protected Split file or remove protection
Connector sync reprocesses unchanged files Incremental sync disabled Set SENTINEL_CONNECTORS_INCREMENTAL_SYNC_ENABLED=true
Legacy connector sources are not pruned Pre-existing chunks missing connector metadata Run legacy migration: enable SENTINEL_CONNECTORS_LEGACY_MIGRATION_ENABLED=true (dry-run first)
Response shows gibberish characters or duplicated clauses Corrupted/binary-like text leaked into synthesized output Treat as FAIL in UAT; verify latest build with formatting sanitizer and re-run response-format checks

Document Issues

  • Password-protected PDFs: Remove protection before upload
  • Scanned PDFs: Limited OCR support, quality varies
  • Large files: Split into smaller documents (<50MB)
Critical Error Support

If issues persist, generate a system log and contact your Deployment Specialist. Include: error messages, timestamp, user ID, and reproduction steps.
